# SUI Move Contract Audit — 2026-06-04

In-session review of all 10 Move contracts in `contracts/sui/sources/`,
triggered by the operator report "withdrawal lower than expected for some
users" on 2026-06-03. Not a substitute for external review (Movebit /
OtterSec / Halborn) — this captures critical + high findings the
in-session pass surfaced + the fixes shipped.

## Severity scale

- **CRITICAL** — direct fund loss or contract bricking, exploitable today
- **HIGH** — fund-at-risk path or invariant violation requiring narrow conditions
- **MEDIUM** — degraded semantics, surprising behavior, future-scaling concern
- **LOW** — code quality, minor info disclosure

## Findings + fixes

### CRITICAL — community_pool_usdc.move share-math ignored off-chain assets

**Status:** fixed in commits `e8942f76` (oracle field + cron attestation)
and `252a4284` (open_hedge ratio rewire).

`calculate_assets_for_shares`, `calculate_shares_for_deposit`, and
`get_nav_per_share` used only `balance::value(&state.balance) +
VIRTUAL_ASSETS` as total assets — ignoring ~$44 of wBTC/wETH/SUI held on
the admin wallet and BlueFin collateral.

Live impact 2026-06-03: balance $0.41 vs true NAV $44.99. A member
withdrawing 3 shares (10% of pool) would receive $0.135 instead of $4.50.
Symmetric over-issuance on deposit.

Fix: added external NAV oracle via dynamic_field (upgrade-safe — no
struct field added). `admin_attest_external_nav(AdminCap, value, clock)`
with 30% per-call change cap. `admin_set_external_nav_required(bool)`
gates strict-mode freshness assert. Math functions use new internal
`total_assets_including_external`. Cron pushes external NAV every tick
via `attestExternalNav()` in sui-community-pool/route.ts.

### CRITICAL — community_pool_usdc.move close_hedge accepted unverified PnL

**Status:** fixed in commit `252a4284`.

Original `close_hedge` took `pnl_usdc + is_profit + funds` as inputs and
joined whatever funds it got into the pool, regardless of consistency.
A compromised AgentCap holder could call with `is_profit=false,
pnl_usdc=collateral, funds=Coin::zero()`, removing the hedge from
active_hedges and decrementing total_hedged_value without returning any
USDC. Silent drain proportional to any open hedge on-chain.

Fix: assert `coin::value(&funds) >= expected_return` where expected =
`is_profit ? collateral + pnl : max(0, collateral - pnl)`. Generous
returns (e.g. funding rewards captured during hedge life) still accepted;
excess goes into pool balance.

### CRITICAL — community_pool.move close_pool_hedge same drain vector

**Status:** fixed in commit (this commit).

Same shape of bug in the SUI-native sibling. CLAUDE.md flags the USDC
pool as the production target, but if the SUI-native pool is still
hot-callable, the AgentCap drain works identically.

Fix: same funds-verify assert added to `close_pool_hedge`.

### HIGH — community_pool_usdc.move open_hedge ratios broke after external-NAV fix

**Status:** fixed in commit `252a4284`.

Yesterday's `get_total_nav` change to include external_nav cascaded into
`open_hedge`'s reserve-ratio, max-hedge-ratio, and daily-hedge-cap
checks. At $0.41 on-chain vs $44 external, the 20% reserve floor became
$8.89 — unreachable on-chain. Every cron auto-hedge tick would have
reverted with `E_RESERVE_RATIO_BREACHED`.

Fix: new `get_onchain_nav()` = `balance + hedge_state.total_hedged_value`
used by the open_hedge ratio checks (which are about the contract's
controllable USDC, not depositor wealth). Share-pricing math still uses
the full `get_total_nav()`.

### HIGH — community_pool_usdc.move emergency_withdraw FCFS race

**Status:** fixed in commit `252a4284`.

When the pool is paused or circuit-tripped, members can call
`emergency_withdraw`. Original logic capped payout at
`min(fair_share_value, available_balance)`. First member to call could
drain the balance up to their full fair share, leaving later members
with 0. Severity increased after the external-NAV fix because fair share
value now often exceeds available balance.

Fix: pro-rata cap. `amount = min(fair_share, available × shares /
total_shares)`. Every member gets a proportional cut regardless of call
order. Defensive guard on `amount > 0` around the transfer.

### CRITICAL — hedge_executor.move oracle manipulation by AdminCap

**Status:** fixed in commit (this commit). Contract is testnet-only today
per the deploy commit (`c2205326`).

`update_price_feed` accepted any positive price under AdminCap. No
bounded change, no timestamp, no anti-manipulation. Admin could push
arbitrary price (e.g. drop BTC to $1) between user transactions, causing
`close_hedge` to compute massive PnL either direction and drain the
collateral pool. Single-block rug.

Fix: bound per-update change to 50% of prior value. Still permissive
enough for legitimate volatility (BTC's worst day in 2020 was -45%) but
stops the single-block rugpull. Long-term move to Pyth oracle integration.

### HIGH — community_pool.move shares math ignores hedge state

**Status:** NOT FIXED. Flagged for the same upgrade window as
community_pool_usdc.

community_pool.move (SUI-native, deprecated per CLAUDE.md but possibly
still hot-callable) has the same balance-only share math. `get_total_nav`
there only returns `balance::value(&state.balance)` — even less than
the USDC pool's pre-fix version, which at least included
`hedge_state.total_hedged_value`. If SUI native pool has any off-chain
positions, withdrawing members are underpaid.

Recommended fix: same external-NAV oracle pattern applied to USDC pool.
~150 lines. Defer until/unless the SUI-native pool is confirmed to have
real members + off-chain positions; the USDC pool is the live target.

## Findings observed but not patched in this pass

### MEDIUM — fee math overflow above $500M NAV

`collect_management_fee_internal` computes `(nav * fee_bps *
time_elapsed_sec) / (BPS_DENOMINATOR * SECONDS_PER_YEAR)`. At $500M NAV
× 50 bps × 1 day = 2.16e18, within u64 (1.8e19) but tight. A week of
skipped fee collection at $500M = overflow.

Already mitigated by `NAV_SAFETY_CEILING_USDC` halt and the daily
`sui-collect-fees` cron heartbeat. Long-term: u128 math redeploy
(T4-B in the pre-deposit TODO).

### MEDIUM — performance fee can be inflated via attested external NAV

After the 2026-06-03 oracle fix, the admin-attested external NAV feeds
into `get_nav_per_share`, which feeds into `collect_performance_fee_internal`'s
high-water-mark check. An admin who slightly inflates external NAV
attestation would trigger HWM crossings + performance fee accrual.

Mitigations in place: 30% per-call change cap, fees go to treasury (also
admin-controlled), so this is transparency rather than theft. Long-term:
multi-attestor consensus or Pyth-backed valuation oracle.

### MEDIUM — open_hedge / close_hedge race window

The cron calls `open_hedge` then off-chain swaps USDC to wBTC. Between
the on-chain tx and the off-chain settlement, NAV is misreported as
"hedge_state value moved to admin" while no real off-chain asset exists
yet. Window is ~1-5 seconds in normal operation but could be exploited
by a sandwicher in pathological conditions.

Mitigation: cron operates with sub-second cadence on the swap leg; pool
caps individual hedge size; this is acceptable for current volumes.

### LOW — admin_reset_hedge_state emits UsdcPoolPaused event for non-pause action

`admin_reset_hedge_state` and `admin_reset_daily_hedge` both emit
`UsdcPoolPaused { paused: false }` events as "signal state change"
proxies. Indexers and event subscribers parsing pause events will see
false positives. Recommend dedicated event types.

### LOW — emergency_withdraw doesn't decrement member_count

After full withdrawal in `emergency_withdraw`, `member.shares` is set
to 0 but the member row stays in the table and `member_count` is not
decremented. `get_pool_stats` will report stale member count after an
emergency drain. Cosmetic — does not affect correctness.

   upgrade payload — apply them together.

2. **Engage external auditor for the 7 unreviewed contracts.** Budget

3. **Move `AdminCap` to multi-sig** (already T4-A). Reduces the impact

   adding a quorum requirement.

   Reduces trust surface from "admin can rugpull via stale/false price"
   to "Pyth must remain functional."
