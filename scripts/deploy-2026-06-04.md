## Blockers found 2026-06-04

When attempting the live deploy, three issues stacked up that made it
unwise to proceed:

1. **SUI CLI out of date.** Installed `sui 1.69.2` supports protocol
   v120; mainnet is on v124. `sui client upgrade --dry-run` panics
   client-side with `Network protocol version is ProtocolVersion(124),
   but the maximum supported version by the binary is 120`. The real
   upgrade path *might* still work since the network validates with
   its own protocol config — but dep verification can fail and we
   can't get a real gas estimate without the dry-run path.
   **Fix:** `suiup install sui` (or download the latest from
   github.com/MystenLabs/sui/releases) before next attempt.

2. **Deployer gas balance is 0.063 SUI — REAL upgrade cost is ~0.5 SUI.**

   Revised 2026-06-05 after investigating "InsufficientGas" failures
   with the new CLI. Looking at the previous successful upgrade tx
   (`47gd2tp8CGTiZdfKuFKaYYMxt5p3qmPZCfKadJHJPeDW`):

   ```
   budget set:    500,000,000 MIST (0.5 SUI)
   storage cost:  454,578,800 MIST (0.455 SUI) ← the dominant cost
   computation:     2,741,080 MIST
   actual burn:   454,724,100 MIST (~0.455 SUI)
   ```

   Publishing 55 KB of new package bytecode at 76 MIST/byte produces
   ~0.42 SUI of storage cost alone, plus computation. Our failed
   dry-runs were reporting a misleading "estimated cost = 1.6M MIST"
   because that number is the cost-up-to-failure-point, not the true
   upgrade cost.

   **The deployer needs ≥0.6 SUI for the upgrade alone**, plus
   ~0.05 for pause+prover keys+strict+unpause. Recommend topping up
   to at least **1 SUI** for the deploy chain plus future ops.

   Deployer address: `0x99a3a0fd45bb6b467547430b8efab77eb64218ab098428297a7a3be77329ac93`.

3. **`pause-sui-pool.ts` requires `SUI_POOL_ADMIN_KEY` in env.** The
   local `.env.local` has only DB + QStash + Crypto.com creds; no SUI
   admin key. Two options:
   - Pull `SUI_POOL_ADMIN_KEY` from Vercel into `.env.local` for the
     pause window only, then delete after unpause.
   - OR pause via direct `sui client call community_pool_usdc::set_paused`
     since the deployer wallet IS the admin. Bypasses script entirely.

When all three are resolved, the rest of this runbook proceeds as
written. IDs are already verified and captured below.

## Step 4 — (NEW today) set the ed25519 prover pubkey

This activates the real ZK verification across `zk_verifier`,
`zk_proxy_vault`, `zk_hedge_commitment`, `rwa_manager`. Until set,
those contracts fall back to the (insecure) legacy length check.

Generate or fetch the prover's ed25519 public key (32 bytes, hex).
If the Python prover at `zkp/api/server.py` doesn't have a signing key
yet, generate one:

```bash
python -c "
import nacl.signing, binascii
sk = nacl.signing.SigningKey.generate()
print('private (KEEP SAFE):', binascii.hexlify(bytes(sk)).decode())
print('public (push on-chain):', binascii.hexlify(bytes(sk.verify_key)).decode())
"
```

Store the private key in the prover's env (`ZK_PROVER_SIGNING_KEY`).
Push the 32-byte public key on-chain:

```bash
PROVER_PUBKEY_HEX=...   # 64 hex chars, NO 0x prefix

# Set for each of the 4 contracts using the appropriate State object.
# For now ONLY zk_verifier is in active use; the others are dormant.
# Setting them all keeps the contracts secure-by-default if anyone
# starts calling them later.

sui client call \
  --package $NEW_PACKAGE_ID \
  --module zk_verifier \
  --function admin_set_prover_pubkey \
  --args $ADMIN_CAP_ID $ZK_VERIFIER_STATE_ID "[0x$PROVER_PUBKEY_HEX]" \
  --gas-budget 10000000
```

(skip the other 3 contracts if their state objects don't exist on
mainnet — they're dormant per CLAUDE.md)
